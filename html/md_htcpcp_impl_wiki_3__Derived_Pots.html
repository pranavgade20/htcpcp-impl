htcpcp implementation
The 2 most important types of pots to be implemented are a coffee pot and a tea pot

Coffee Pot:

Requirements:

  • Accept requests to start brewing coffee
  • Take up a cup of plain coffee and add to it milk, sweetener, syrup, spice, alcohol as requested by the client
  • Keep brewing the cup of coffee until receipt of command to stop brewing, and then remove the ready cup of coffee from itself.

How Coffee Pot Satisfies the Requirements:

  • The CoffeePot class extends the Pot class by inheriting it, hence provides the same functionality, and more
  • Sets the server name to "Coffee Pot", according to which the brew method sets appropriate response headers (Server: Coffee Pot)

Tea Pot:

Requirements:

  • Respond to every POST or BREW method request with Error 418 I'm a Teapot (to signal that coffee can not be brewed by a tea pot)

How Tea Pot Satisfies the Requirements:

  • The TeaPot class extends and overrides parts of the Pot class by inheriting it, hence provides same functions (names) but different functionality
  • Sets the server name to "Tea Pot", according to which the brew method sets appropriate response headers (Server: Tea Pot)
  • Overrides all the add[Addition]() methods to disallow addition and instead throw an error with integer code 418. This error is caught in the brew method and accordingly a response object is returned
